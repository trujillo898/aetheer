--- D006 ---
DECISIÓN: TradingView MCP como fuente primaria de datos de mercado
FECHA: abril 2026
CONTEXTO: sin acceso a OANDA API ni a ningún feed institucional de precio.
  Fuentes actuales: Alpha Vantage (25 calls/día, delay), scraping (frágil).
  tradingview-mcp (github.com/tradesdontlie/tradingview-mcp) conecta Claude Code
  con TradingView Desktop vía Chrome DevTools Protocol. 78 tools.
  TV es la única fuente de precio en tiempo real con calidad de gráfico.
  Entorno: Ubuntu Server 24.04 con Hyprland (Wayland), TV Desktop instalado.
  El sistema puede auto-lanzar y auto-reiniciar TV por comando.
OPCIONES:
  A) TV MCP como fuente primaria + sistema anti-fragilidad de 5 capas
     — pros: dato idéntico al gráfico, yields/VIX/oro/DXY directo, indicadores técnicos,
       auto-launch elimina dependencia operacional del usuario
     — contras: interfaz no documentada, depende de TV Desktop funcional
  B) No integrar, seguir con Alpha Vantage + scraping
     — pros: sin dependencia nueva
     — contras: 25 calls/día, sin DXY directo, sin indicadores, sin yields real-time
DECIDIDO: A — sin OANDA, TV es la única fuente viable de precio real-time.
  La fragilidad se mitiga con sistema de 5 capas: heartbeat, cache/snapshot,
  reconexión auto, degradación explícita, auto-launch/restart.
  Cascada: TV MCP → cache/snapshot → Alpha Vantage → scraping.
IMPACTO:
  - TV MCP es fuente primaria de precio, DXY, yields, correlaciones, indicadores
  - Sistema anti-fragilidad de 5 capas:
    1. Heartbeat cada 60s vía tv_health_check
    2. Snapshot en SQLite en cada lectura exitosa (precio, yields, correlaciones)
    3. Reconexión automática con backoff si TV no responde
    4. Degradación a Operating Mode DEGRADED (no fallo) si TV no disponible
    5. Auto-launch de TV Desktop en bootstrap + auto-restart si PID muere
  - Bootstrap unificado: un solo comando arranca TV + MCP servers + heartbeat
  - Alpha Vantage pasa a fallback
  - Scraping pasa a último recurso
  - Cross-validation: TV vs Alpha Vantage cuando ambas disponibles
  - Capacidad nueva: indicadores técnicos directo del gráfico (ATR, RSI, EMA)
  - Operating Modes basados en disponibilidad de TV:
    TV activo + datos frescos → FULL
    TV crasheó, auto-restart en curso → DEGRADED transitorio (~30s)
    TV no puede arrancar → DEGRADED persistente (Alpha Vantage + scraping)
    Todo falla → KILL SWITCH / OFFLINE
RIESGOS ACEPTADOS:
  - Interfaz no documentada puede romperse con actualización de TV
  - Mitigación: pinear versión TV Desktop, monitorear repo del MCP, fork si muere
  - ToS de TradingView pueden restringir uso automatizado
  - Mitigación: uso local, no redistribución, no comercial
IMPLEMENTACIÓN ACTUAL: agregar tradingview-mcp como MCP server,
  implementar heartbeat + cache/snapshot en price-feed MCP,
  agregar auto-launch de TV al bootstrap script,
  agregar Operating Modes basados en disponibilidad de TV
IMPLEMENTACIÓN IDEAL: Price Agent con adapter pattern,
  Data Integrity Agent con cross-validation automática,
  Operating Modes dinámicos con transición automática
ESTADO: implementado
